# Создание формы интерфейса в 1С:Элемент

## Шаг 0: Определи что нужно создать

Из запроса пользователя определи:

| Что создать | Когда |
|-------------|-------|
| `ФормаОбъекта` | Форма редактирования одной записи |
| `ФормаСписка` | Форма со списком записей |
| Обе формы | Если пользователь хочет полный интерфейс (типичный случай) |

Уточни у пользователя имя объекта (Справочника или Документа), если оно не указано.

## Шаги 1 и 2: выполни параллельно

**Шаг 1 — Загрузи спецификацию.** Прочитай нужные файлы из `references/`:

- При создании `ФормаОбъекта` → прочитай `references/ФормаОбъекта.md`
- При создании `ФормаСписка` → прочитай `references/ФормаСписка.md`
- Всегда дополнительно прочитай `references/elements.md` — справочник элементов UI

**Шаг 2 — Разведка проекта.** Используй скилл `xbsl-explore` с именем объекта.

Из результата разведки:
- Возьми `suggested_path` — папка куда класть файлы формы
- Найди `.yaml` файл объекта в `suggested_path` и **прочитай его** — нужны реквизиты (поля) объекта для генерации привязок `=Объект.Поле`

## Шаг 3: Сгенерируй UUID

Нужен **1 UUID на каждую форму**. Вызови скилл `xbsl-uuid` с нужным числом (1 или 2).

## Шаг 4: Создай файлы

Для каждой создаваемой формы:

- Создай `{ИмяОбъекта}ФормаОбъекта.yaml` или `{ИмяОбъекта}ФормаСписка.yaml` в `suggested_path`
- Создай `.xbsl` файл **только если** пользователь явно запросил обработчики

### Именование файлов

| Тип формы | Имя файла |
|-----------|-----------|
| ФормаОбъекта для `Сотрудники` | `СотрудникиФормаОбъекта.yaml` |
| ФормаСписка для `Сотрудники` | `СотрудникиФормаСписка.yaml` |
| ФормаОбъекта для `Заказы` | `ЗаказыФормаОбъекта.yaml` |
