# Создание формы интерфейса в 1С:Элемент

## Шаг 0: Определи что нужно создать

| Что создать | Когда |
|-------------|-------|
| `ФормаОбъекта` | Форма редактирования одной записи |
| `ФормаСписка` | Форма со списком записей |
| Обе формы | Если нужен полный интерфейс (типичный случай) |

Уточни у пользователя имя объекта, если оно не указано.

## Шаги 1 и 2: выполни параллельно

**Шаг 1 — Загрузи спецификацию.** Прочитай нужные файлы из `references/`:

- При создании `ФормаОбъекта` → прочитай `references/ФормаОбъекта.md`
- При создании `ФормаСписка` → прочитай `references/ФормаСписка.md`
- Читай `references/elements.md` **только если** пользователь упоминает нестандартные элементы: вложенные файлы, изображения, сворачиваемые блоки, статусы с цветом — иначе пропусти

**Шаг 2 — Разведка проекта.** Используй скилл `xbsl-explore` с именем объекта.

Из результата:
- Возьми `suggested_path` — папка куда класть файлы формы
- Найди `.yaml` файл объекта в `suggested_path` и **обязательно прочитай его**

> Имена и типы полей — только из YAML объекта. Никогда не придумывай имена самостоятельно: поле называется `Покупатель`, а не `Контрагент` — потому что так написано в YAML объекта. Это главная причина ошибок без этого шага.

## Шаг 3: Сгенерируй UUID

Нужен **1 UUID на каждую форму**. Вызови скилл `xbsl-uuid` с нужным числом (1 или 2).

## Шаг 4: Создай файлы

Для каждой создаваемой формы создай файл в `suggested_path`.

### Именование файлов

| Тип формы | Имя файла |
|-----------|-----------|
| ФормаОбъекта для `Сотрудники` | `СотрудникиФормаОбъекта.yaml` |
| ФормаСписка для `Сотрудники` | `СотрудникиФормаСписка.yaml` |

### Заголовок формы

- `ФормаОбъекта` → единственное число: `Сотрудник`, `Заказ покупателя`, `Контрагент`
- `ФормаСписка` → множественное/название объекта: `Сотрудники`, `Заказы покупателей`

### Важные правила

- `ВключатьВАвтоИнтерфейс: Ложь` — **всегда** указывай в обеих формах
- Для ФормаСписка: динамический список описывается в секции `Свойства` внизу файла (не внутри `Содержимое`), таблица ссылается на него через `Источник: =Список`
- Колонки в ФормаСписка используют `Значение: =ДанныеСтроки.Данные.<Поле>` (не `ПолеЗначения`)
- Для Справочника без ТЧ — шаблон `ПроизвольныйШаблонФормы`
- Для Документа или объекта с ТЧ — шаблон `ШаблонФормыСРазделами`
- `.xbsl` файл создавай **только** если пользователь явно запросил обработчики
