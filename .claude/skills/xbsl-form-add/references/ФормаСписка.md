# Спецификация: ФормаСписка

## Структура YAML

> `<ns>` — значение `namespace` из скрипта (`vendor::project::subsystem`)

```yaml
ВидЭлемента: КомпонентИнтерфейса
Ид: <UUID>
Имя: <ИмяОбъекта>ФормаСписка
ОбластьВидимости: ВПодсистеме
Наследует:
    Тип: ФормаСписка
    ВключатьВАвтоИнтерфейс: Ложь
    Заголовок: <Заголовок>
    ДополнительныеКоманды:
        Тип: ФрагментКомандногоИнтерфейса
        Элементы:
            - =Обновить
    КомандыСоздания: =Компоненты.ОсновнаяТаблица.ДобавитьСтроку
    КомпонентТаблицы: =Компоненты.ОсновнаяТаблица
    Содержимое:
        Тип: ПроизвольныйШаблонФормы
        Содержимое:
            Тип: Таблица<ДинамическийСписок<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>>
            Имя: ОсновнаяТаблица
            Источник: =Список
            Колонки:
                -
                    Тип: СтандартнаяКолонкаТаблицы<СтрокаДинамическогоСписка<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>>
                    Имя: <Поле1>
                    Значение: =ДанныеСтроки.Данные.<Поле1>
                -
                    Тип: СтандартнаяКолонкаТаблицы<СтрокаДинамическогоСписка<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>>
                    Имя: <Поле2>
                    Значение: =ДанныеСтроки.Данные.<Поле2>
            КомандыСтроки:
                Тип: ФрагментКомандногоИнтерфейса<КомандаСПараметром<СтрокаДинамическогоСписка<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>>>
                Элементы:
                    - =Компоненты.ОсновнаяТаблица.СоздатьКопию
Свойства:
    -
        Имя: Список
        Тип: ДинамическийСписок<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>
        ЗначениеПоУмолчанию:
            ИмяТипаДанныхСтроки: ДанныеСтрокиСписка
            ОсновнаяТаблица:
                Таблица: <ИмяОбъекта>
            Поля:
                -
                    Тип: ПолеДинамическогоСписка
                    Выражение: <Поле1>
                -
                    Тип: ПолеДинамическогоСписка
                    Выражение: <Поле2>
```

## Правила

- `ВключатьВАвтоИнтерфейс: Ложь` — обязательно всегда
- `<ns>` — полный namespace из скрипта (`namespace`), например `korolevpavel::cc_1c_xbsl_test::Основное`
- `КомандыСоздания: =Компоненты.ОсновнаяТаблица.ДобавитьСтроку` — всегда именно так
- `Источник: =Список` — таблица ссылается на свойство `Список`, определённое в `Свойства`
- Колонки: `Значение: =ДанныеСтроки.Данные.<Поле>` — так привязываются данные строки
- `Имя` колонки — произвольное, обычно совпадает с именем поля
- Поля в `Свойства.ЗначениеПоУмолчанию.Поля` — должны совпадать с полями в колонках

### Выбор колонок

Добавляй **все поля объекта** как колонки. Для Справочника первая колонка — всегда `Наименование`, затем остальные поля в порядке из YAML объекта.

## Сортировка

```yaml
            ЗначениеПоУмолчанию:
                ...
                Сортировка:
                    -
                        Поле: <ПолеСортировки>
                        НаправлениеСортировки: ПоУбыванию   # или ПоВозрастанию
```

Документы обычно сортируются по `Дата` возрастанию. Справочники — по `Наименование` возрастанию.

## Колонка со статусом (перечисление с цветом)

```yaml
                -
                    Тип: СтандартнаяКолонкаТаблицы<СтрокаДинамическогоСписка<<ns>::<ИмяОбъекта>ФормаСписка.ДанныеСтрокиСписка>>
                    Имя: Статус
                    Значение: =ДанныеСтроки.Данные.Статус
                    Вид: Значок
                    Цвет: =ДанныеСтроки.Данные.Статус.ПолучитьЦвет()
```

## Действия над отмеченными строками

```yaml
            ИспользоватьОтметкиСтрок: Истина
            КомандыНадОтмеченнымиСтроками:
                Тип: ФрагментКомандногоИнтерфейса<КомандаСПараметром<Массив<<ИмяОбъекта>.Ссылка>>>
                Элементы:
                    -
                        Тип: КомандаСПараметром<Массив<<ИмяОбъекта>.Ссылка>>
                        Представление: <ТекстКоманды>
                        Изображение: <Иконка>.svg
                        Обработчик: <ИмяОбработчика>
```
