## Отличие от стандартной ФормаСписка

Стандартная форма списка (`xbsl-form-add`) создаёт `Таблица<ДинамическийСписок>` — строчный список.
Этот скилл создаёт `ПроизвольныйСписок<ДинамическийСписок>` с `КонтейнерСтрок: Матричная` —
сетку карточек. Требует ДВУХ файлов: форма + компонент строки (`СтрокаСписка<Объект>.yaml`).

Прочитай `references/patterns.md` перед выполнением — там YAML-шаблоны обоих файлов.

## Алгоритм

### Шаг 1 — Анализ объекта

```bash
python3 .claude/skills/xbsl-form-info/scripts/form_info.py --name <ИмяОбъекта> --root .
```

Из результата взять:
- `object_path` — папка для создания файлов
- `object_file` — файл объекта для обновления `Интерфейс`
- `object_type` — тип объекта (Справочник, Документ и т.д.)
- `namespace` — пространство имён (`vendor::project::subsystem`)
- `fields` — список реквизитов с именами и типами
- `existing_forms["ФормаСписка"]` — существует ли форма списка

Если скрипт вернул `error` — сообщи пользователю и остановись.

Если `existing_forms["ФормаСписка"]` не пустой — спроси пользователя:
- **Заменить** → продолжить (перезаписать файлы)
- **Отмена** → остановиться

### Шаг 2 — Определить поля карточки

Из `fields` выбрать:

| Роль | Как выбирать |
|------|-------------|
| **Заголовок** (`Заголовок:`) | Первое поле типа `Строка` с именем `Наименование`, `Заголовок`, `Название` или любое первое строковое поле |
| **Содержимое** (`Содержимое:`) | Поля `ДатаВремя`, `Дата`, `Число`, `Строка`, ссылки — исключая заголовок. `Строка`/`Дата` — прямое выражение; ссылки (`*.Ссылка?`) — через `Надпись`; несколько полей — `Группа` с `Надписями` |
| **Изображение** (`Изображение:`) | Только если есть поле типа `ДвоичныйОбъект.Ссылка?` — добавить в карточку |

Если вторичных полей нет — `Содержимое` карточки опустить.

### Шаг 3 — UUID

Вызвать скилл `xbsl-uuid` → **2 UUID**: один для ФормаСписка, один для СтрокаСписка.

### Шаг 4 — Создать `<Объект>ФормаСписка.yaml`

Файл создаётся в `object_path`. Шаблон из `references/patterns.md`, раздел «Файл 1».

Имя компонента строки: `СтрокаСписка<Объект>` (например `СтрокаСпискаЗадачи`).

### Шаг 5 — Создать `СтрокаСписка<Объект>.yaml`

Файл создаётся в `object_path` рядом с формой.

**Выбор шаблона:**

| Ситуация | Шаблон |
|----------|--------|
| Нет поля `ДвоичныйОбъект.Ссылка?` | `СтандартнаяКарточка` — раздел «Файл 2» |
| Есть поле `ДвоичныйОбъект.Ссылка?` | `ПроизвольнаяКарточка` — раздел «Карточка с фото» |

При `ПроизвольнаяКарточка`:
- `Компоновка: Вертикальная` на `Группа` — обязательно (иначе элементы горизонтально)
- `МинимальнаяШирина` в `КонтейнерСтрок`: 250 для людей (портреты), 400 для остальных
- Поле фото обязательно включить в `Источник.Поля` формы списка

### Шаг 6 — Обновить объектный YAML

Добавить или обновить секцию `Интерфейс` в `object_path/object_file` — по шаблону из `references/patterns.md`, раздел «Обновление объектного YAML».

- `ИспользоватьСозданиеПриВводе: Истина` — только для `Справочник`
- Если уже есть `Объект.Форма` — сохранить, добавить только `Список.Форма`

### Шаг 7 — Итог

Перечислить созданные/изменённые файлы:
- `<Объект>ФормаСписка.yaml` — создана форма карточек
- `СтрокаСписка<Объект>.yaml` — создан компонент строки
- `<ОбъектYAML>` — обновлён `Интерфейс.Список.Форма`
