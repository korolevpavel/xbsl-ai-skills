# Паттерны xbsl-form-cards

Шаблоны основаны на рабочем примере `tools/vendor/pp/ДинамическийСписокВВидеКарточек/`.

---

## Файл 1: `<Объект>ФормаСписка.yaml`

```yaml
ВидЭлемента: КомпонентИнтерфейса
Ид: <UUID-1>
Имя: <Объект>ФормаСписка
ОбластьВидимости: ВПодсистеме
Наследует:
    Тип: ФормаСписка
    ВключатьВАвтоИнтерфейс: Ложь
    Заголовок: <ЗаголовокФормы>
    ДополнительныеКоманды:
        Тип: ФрагментКомандногоИнтерфейса
        Элементы:
            - =Обновить
    КомандыСоздания: =Компоненты.ОсновнаяТаблица.ДобавитьСтроку
    КомпонентТаблицы: =Компоненты.ОсновнаяТаблица
    Содержимое:
        Тип: ПроизвольныйШаблонФормы
        Содержимое:
            Тип: Группа
            Содержимое:
                -
                    Тип: ПроизвольныйСписок<ДинамическийСписок<<namespace>::<Объект>ФормаСписка.ДанныеСтрокиСписка>>
                    Имя: ОсновнаяТаблица
                    ОбрабатыватьНажатие: Истина
                    ТипКомпонентаСтроки: СтрокаСписка<Объект>
                    Источник:
                        ИмяТипаДанныхСтроки: ДанныеСтрокиСписка
                        ОсновнаяТаблица:
                            Таблица: <Объект>
                        Поля:
                            -
                                Тип: ПолеДинамическогоСписка
                                Выражение: Ссылка
                            -
                                Тип: ПолеДинамическогоСписка
                                Выражение: <ОсновноеПоле>
                            # добавить остальные поля карточки
                    КонтейнерСтрок:
                        Тип: Группа
                        Имя: МатричнаяГруппа
                        Компоновка: Матричная
                        РастягиватьПоВертикали: Ложь
                        РастягиватьПоГоризонтали: Истина
                        НастройкиМатричнойКомпоновки:
                            АвтоЗаполнение: ДобавлятьКолонкиИСтроки
                            ОписаниеАвтоматическихКолонок:
                                МинимальнаяШирина: 400
                                РастягиватьПоГоризонтали: Истина
                            ОписаниеАвтоматическихСтрок:
                                РастягиватьПоВертикали: Истина
```

**Ключевые переменные:**
- `<namespace>` — из `xbsl-form-info`, например `Demo::TestApp::Основное`
- `<Объект>` — имя справочника/документа, например `Задачи`
- `<ЗаголовокФормы>` — заголовок формы во множественном числе, например `Задачи`
- `<ОсновноеПоле>` — первичное строковое поле (Наименование, Заголовок и т.п.)
- `ТипКомпонентаСтроки` — имя второго файла (без расширения), например `СтрокаСпискаЗадачи`

## Файл 2: `СтрокаСписка<Объект>.yaml`

Имя файла: `СтрокаСписка<Объект>.yaml`, например `СтрокаСпискаЗадачи.yaml`.

```yaml
ВидЭлемента: КомпонентИнтерфейса
Ид: <UUID-2>
Имя: СтрокаСписка<Объект>
ОбластьВидимости: ВПодсистеме
Наследует:
    Тип: ПроизвольнаяСтрокаСписка<СтрокаДинамическогоСписка<<namespace>::<Объект>ФормаСписка.ДанныеСтрокиСписка>>
    Содержимое:
        Тип: СтандартнаяКарточка
        Заголовок: =ДанныеСтроки.Данные.<ОсновноеПоле>
        РастягиватьПоВертикали: Истина
        РастягиватьПоГоризонтали: Истина
        Содержимое: =ДанныеСтроки.Данные.<ВторичноеПоле>
```

**Варианты `Содержимое` карточки:**

`Содержимое` в `СтандартнаяКарточка` принимает `Строка | Компонент`. Правила:

```yaml
# Строковое поле (Тип: Строка) — прямое выражение:
Содержимое: =ДанныеСтроки.Данные.Описание

# Дата/время — с форматированием:
Содержимое: =ДанныеСтроки.Данные.Дата.Представление("дд ММММ гггг ЧЧ:мм")

# Ссылка (Справочник.Ссылка?) — ОБЯЗАТЕЛЬНО через Надпись, не напрямую:
Содержимое:
    Тип: Надпись
    Значение: =ДанныеСтроки.Данные.Проект

# Несколько полей — через Группу:
Содержимое:
    Тип: Группа
    Содержимое:
        -
            Тип: Надпись
            Значение: =ДанныеСтроки.Данные.<Поле1>
        -
            Тип: Надпись
            Значение: =ДанныеСтроки.Данные.<Поле2>

# Нет вторичных полей — опустить Содержимое целиком
```

**Если есть фото (`ДвоичныйОбъект.Ссылка?`):**

> ⚠️ `СтандартнаяКарточка.Изображение` отображает фото как большой баннер без ограничения высоты.
> Используй его только для небольших иконок/аватаров.
> Для объектов, где фото — не главный элемент (задачи, товары и т.д.), **не добавляй `Изображение`** в карточку.
> Если нужен аватар-thumbnail — используй `Изображение` только когда это логично (сотрудники, контакты).

## Обновление объектного YAML

В секцию `Интерфейс` объекта добавить/обновить:

```yaml
Интерфейс:
    ВключатьВАвтоИнтерфейс: Истина
    ИспользоватьСозданиеПриВводе: Истина   # только для Справочника
    Список:
        Форма: <Объект>ФормаСписка
```

Если уже есть `Объект.Форма` — сохранить, добавить только `Список.Форма`.
