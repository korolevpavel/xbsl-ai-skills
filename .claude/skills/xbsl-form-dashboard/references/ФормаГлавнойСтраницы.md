# Спецификация: ФормаГлавнойСтраницы

Шаблоны основаны на рабочих примерах:
- `tools/vendor/CRM/Основное/ГлавнаяСтраница.yaml` — KPI, таблицы, диаграммы
- `tools/vendor/pp/ПроизвольноеКлиентскоеПриложение/Главная.yaml` — баннер, карусель, бенто
- `tools/vendor/pp/Баннер/Главная.yaml` — баннер-карточки, карусель отзывов, ПоКолонкам
- `tools/vendor/pp/НесколькоКлиентскихПриложений/ГлавнаяСтраницаЗаказчика.yaml` — поиск, бенто
- `tools/vendor/pp/НесколькоКлиентскихПриложений/ГлавнаяСтраницаИсполнителя.yaml` — статика

---

## Базовая структура YAML

```yaml
ВидЭлемента: КомпонентИнтерфейса
Ид: <UUID>
Имя: ГлавнаяСтраница
ОбластьВидимости: ВПодсистеме
Импорт:
    - <ИмяПодсистемы>          # подсистемы, из которых берутся данные
Наследует:
    Тип: Форма
    ВключатьВАвтоИнтерфейс: Ложь
    ПовторноеИспользованиеФормы: Истина
    Содержимое:
        Тип: ПроизвольныйШаблонФормы
        Содержимое:
            Тип: Группа
            РастягиватьПоВертикали: Ложь
            ШиринаВКолонках: Четверная
            Содержимое:
                # виджеты (карточки) — см. паттерны ниже
```

**Правила:**
- `ВключатьВАвтоИнтерфейс: Ложь` — **всегда**, без исключений
- `ПовторноеИспользованиеФормы: Истина` — **всегда** (кэширование формы)
- `ШиринаВКолонках: Четверная` на корневой Группе — даёт 4-колонную сетку
- `Импорт:` — только если берёте данные из другой подсистемы
- Если нет динамических данных — `Импорт:` можно опустить

## Критические правила (проверено на реальных проектах)

### 1. Статические строки в `Значение:` требуют кавычек

```yaml
# ✅ Правильно:
Значение: "Заказов"
Значение: "В процессе"

# ❌ Ошибка компиляции — платформа ожидает тип или Неопределено:
Значение: Заказов
```

Это касается только статических строковых литералов в `Значение:`. Динамические выражения (`=...`) и ссылки на локализованные строки (`$...`) не требуют кавычек.

### 2. SVG-иконки — только платформенные или из папки Ресурсы/

```yaml
# ✅ Платформенные встроенные иконки (проверены):
Изображение: Плюс.svg
Изображение: Задачи.svg

# ✅ Иконки из папки Ресурсы/ подсистемы:
Изображение: РазделПресейл.svg   # файл в Основное/Ресурсы/

# ❌ Ошибка "Неизвестный ресурс" — таких иконок нет в платформе:
Изображение: Документ.svg
Изображение: Склад.svg
Изображение: Заказы.svg
```

Если нет подходящей встроенной иконки — создайте SVG-файл в папке `<Подсистема>/Ресурсы/` или используйте `Плюс.svg`.

### 3. Видимость объектов из других подсистем

```yaml
# ✅ Доступно из любой подсистемы (при импорте):
ОбластьВидимости: ВПроекте

# ❌ Ошибка "invisible due to visibility modifier @ВПодсистеме":
ОбластьВидимости: ВПодсистеме
```

Если ГлавнаяСтраница в подсистеме `Основное` импортирует `Закупки` и обращается к `ЗаказКлиентаФормаСписка` или `Задачи` — эти объекты в `Закупки` должны иметь `ОбластьВидимости: ВПроекте`.

### 4. Избегайте круговых зависимостей подсистем

Если подсистема `Склад` использует (`Использование: [Основное]`) — то `Основное` не может импортировать `Склад`. Это круговая зависимость. Решения:
- **Перенести** общие данные в нейтральную подсистему
- **Не использовать** типы из подсистемы с обратной зависимостью — убрать `Импорт: Склад` из формы

### 5. Регистрация в навигации — создать ПанельНавигации.yaml при необходимости

Если в проекте нет `ПанельНавигации.yaml` — форма не появится в интерфейсе. Нужно создать:

```yaml
ВидЭлемента: ФрагментКомандногоИнтерфейса
Ид: <UUID>
Имя: ПанельНавигации
ОбластьВидимости: ВПодсистеме
Элементы:
    -
        Тип: НавигационнаяКоманда
        Изображение: Плюс.svg
        Представление: Главная
        ТипФормы: ГлавнаяСтраница
```

### 6. Типизированные структуры для таблиц P3

Тип `<ТипСтруктуры>` в `Таблица<ИсточникДанныхМассив<ТипСтруктуры>>` должен быть определён как `Структура` в той же или доступной (без круговой зависимости) подсистеме. Нельзя ссылаться на структуру из подсистемы с круговой зависимостью.

## Правила компоновки

**ШиринаВКолонках карточки** (внутри корневой Четверной группы):
- `Одинарная` — занимает 1/4 ширины
- `Двойная` — половина ширины (стандарт для большинства карточек)
- `Четверная` — полная ширина (для баннеров)

**МинимальнаяВысота**: `230` — стандарт для KPI-карточек и карточек с таблицами.

**РастягиватьПоГоризонтали: Ложь** на карточке — карточка не растягивается
на всю ширину, используется ШиринаВКолонках для позиционирования.

## P1: KPI-карточка (динамический счётчик)

Карточка с числовым значением, обновляемым при загрузке формы.

```yaml
-
    Тип: СтандартнаяКарточка
    Заголовок: <Название счётчика>
    МинимальнаяВысота: 230
    ОбрабатыватьНажатие: Истина
    ПриНажатии: <ИмяКарточки>ПриНажатии
    РастягиватьПоГоризонтали: Ложь
    ШиринаВКолонках: Двойная
    Картинка:
        Тип: Картинка
        Изображение: <Иконка.svg>
        Цвет: =Цвета.<Цвет>    # Оранжевый | Зеленый | Голубой | Красный
    Команды:
        Тип: ОбычнаяКоманда
        Важность: Обычная
        Видимость: Истина
        Доступность: Истина
        Изображение: Плюс.svg
        Обработчик: Создать<Объект>ПриНажатии
        ОпасностьДействия: Отсутствует
        Представление: <Текст кнопки>
    Содержимое:
        Тип: Группа
        Компоновка: Горизонтальная
        Содержимое:
            -
                Тип: Группа
                ИнтервалМеждуЭлементамиПоВертикали: Отсутствует
                Компоновка: Вертикальная
                РастягиватьПоГоризонтали: Истина
                Содержимое:
                    -
                        Тип: Надпись
                        Важность: Низкая
                        Значение: <Метка1>
                    -
                        Тип: Надпись
                        Имя: НадписьЗначение1           # ← обновляется из .xbsl
                        МаксимальнаяВысотаВСтроках: 1
                        Шрифт: =СтилевыеШрифты.ЗаголовокОбычный
            -
                Тип: Группа
                ИнтервалМеждуЭлементамиПоВертикали: Отсутствует
                Компоновка: Вертикальная
                РастягиватьПоГоризонтали: Истина
                Содержимое:
                    -
                        Тип: Надпись
                        Важность: Низкая
                        Значение: <Метка2>
                    -
                        Тип: Надпись
                        Имя: НадписьЗначение2           # ← обновляется из .xbsl
                        МаксимальнаяВысотаВСтроках: 1
                        Шрифт: =СтилевыеШрифты.ЗаголовокОбычный
```

**Правила:**
- Каждая Надпись со значением **обязана** иметь `Имя:` — иначе её нельзя обновить из .xbsl
- Обращение из .xbsl: `Компоненты.НадписьЗначение1.Значение = ...`
- Одна карточка может содержать несколько пар метка+значение в горизонтальной Группе

## P2: Навигационная карточка (клик → открыть форму)

```yaml
-
    Тип: СтандартнаяКарточка
    Заголовок: <Название раздела>
    МинимальнаяВысота: 230
    ОбрабатыватьНажатие: Истина
    ПриНажатии: Карточка<Раздел>ПриНажатии
    РастягиватьПоГоризонтали: Ложь
    ШиринаВКолонках: Двойная
    Картинка:
        Тип: Картинка
        Изображение: <Иконка.svg>
        Цвет: =Цвета.<Цвет>
    Содержимое:
        Тип: Надпись
        Значение: <Описание раздела>
```

**Обработчик в .xbsl:**
```xbsl
метод Карточка<Раздел>ПриНажатии(Источник: Карточка, Событие: СобытиеПриНажатии)
    <Раздел>ФормаСписка.Открыть()
;
```

## P3: Табличная карточка (данные из массива)

```yaml
-
    Тип: СтандартнаяКарточка
    Имя: Карточка<Имя>
    Заголовок: <Название>
    МинимальнаяВысота: 230
    РастягиватьПоГоризонтали: Ложь
    ШиринаВКолонках: Двойная
    Картинка:
        Тип: Картинка
        Изображение: <Иконка.svg>
        Цвет: =Цвета.<Цвет>
    Команды:
        Тип: ОбычнаяКоманда
        Важность: Обычная
        Видимость: Истина
        Доступность: Истина
        Обработчик: <Действие>ПриНажатии
        ОпасностьДействия: Отсутствует
        Представление: <Текст кнопки>
    Содержимое:
        Тип: Таблица<ИсточникДанныхМассив<<ТипСтруктуры>>>
        Имя: Таблица<Имя>
        ИспользованиеПоиска: Ложь
        ОтображатьНастройки: Ложь
        ОтображатьШапку: Ложь
        ТолькоЧтение: Истина
        Колонки:
            -
                Тип: СтандартнаяКолонкаТаблицы<<ТипСтруктуры>>
                ПолеЗначения: <Поле1>
            -
                Тип: СтандартнаяКолонкаТаблицы<<ТипСтруктуры>>
                ПолеЗначения: <Поле2>
```

**Обновление данных из .xbsl:**
```xbsl
метод ЗаполнитьКарточку<Имя>()
    Компоненты.Таблица<Имя>.Источник.Данные = Получить<Имя>()
;

@НаСервере @ДоступноСКлиента
статический метод Получить<Имя>(): Массив<<ТипСтруктуры>>
    возврат <Объект>.Получить<Имя>()
;
```

## P4: Баннер (картинка + текст + кнопка)

**Простой баннер:**
```yaml
-
    Тип: СтандартнаяКарточка
    ВидОтображения: Баннер
    Заголовок: <Заголовок баннера>
    МинимальнаяВысота: 230
    РасположениеИзображенияВБаннере: ПослеСодержимого   # ДоСодержимого | ПослеСодержимого
    РастягиватьИзображениеВБаннере: Истина
    РастягиватьПоГоризонтали: Ложь
    ШиринаВКолонках: Четверная
    Фон:
        Тип: АбсолютныйЦвет
        Значение: RGB(<HEX>)
    ЦветЗаголовка:
        Тип: АбсолютныйЦвет
        Значение: RGB(<HEX>)
    ЦветСодержимого:
        Тип: АбсолютныйЦвет
        Значение: RGB(<HEX>)
    Картинка:
        Тип: Картинка
        АльтернативныйТекст: <Описание>
        Изображение: <Файл.png>
        Масштабирование: Пропорционально
    Команды:
        Тип: ГруппаКомандногоИнтерфейса
        Элементы:
            -
                Тип: ОбычнаяКоманда
                Важность: Высокая
                Видимость: Истина
                Доступность: Истина
                ОпасностьДействия: Отсутствует
                Представление: <Текст кнопки>
                Обработчик: <Обработчик>
    Содержимое: "<Текст описания>"
    ШрифтЗаголовка:
        Тип: АбсолютныйШрифт
        Полужирный: Истина
        Размер: 36
    ШрифтСодержимого: =СтилевыеШрифты.ОченьКрупный
```

**Правила баннера:**
- `ВидОтображения: Баннер` — обязательно
- `ШиринаВКолонках: Четверная` — баннер обычно на всю ширину
- Фон: предпочитать `=Цвета.Стилевые.ФонПервичный` для брендовых цветов
  или `RGB(...)` для кастомного цвета
- Содержимое баннера — строка (текст) или Надпись

## P5: Диаграмма

```yaml
-
    Тип: СтандартнаяКарточка
    Заголовок: <Название диаграммы>
    РастягиватьПоГоризонтали: Ложь
    ШиринаВКолонках: Двойная
    Содержимое:
        Тип: Группа
        Компоновка: Вертикальная
        ОтступПоГоризонтали: Двойной
        РастягиватьПоГоризонтали: Истина
        Содержимое:
            -
                Тип: <ТипДиаграммы>
                Имя: <ИмяДиаграммы>
```

**Доступные типы диаграмм:**
- `ДиаграммаВоронкаПродаж` — воронкообразная
- `ДиаграммаПродажиПоНаправлениям` — столбчатая по направлениям
- `ДиаграммаКруговая` — круговая
- `ДиаграммаЛинейная` — линейная
- `ДиаграммаСтековая` — стековая
- `ДиаграммаЛепестковая` — лепестковая
- `ДиаграммаСпидометр` — спидометр
- `ДиаграммаВоронкообразная` — воронкообразная (второй вариант)
- `ДиаграммаПузырьковая` — пузырьковая
- `ДиаграммаГанта` — диаграмма Ганта

**Обновление из .xbsl:**
```xbsl
метод ОбновитьДиаграммы()
    Компоненты.<ИмяДиаграммы>.ОбновитьДиаграмму()
;
```

## P6: Карусель

Прокручиваемый горизонтальный ряд элементов (изображений или карточек).

```yaml
-
    Тип: Группа
    Имя: <ИмяКарусели>
    Заголовок: <Подзаголовок секции>  # необязательно
    Компоновка: Карусель
    НастройкиКарусели:
        ИнтервалВоспроизведения: 3с   # автоскролл; убрать если не нужен
        ЦиклическаяНавигация: Истина
        ВидОтображенияКнопок: ВПанели  # Авто | ВПанели — стрелки навигации
    Содержимое:
        -
            Тип: СтандартнаяКарточка
            РастягиватьПоВертикали: Истина
            Фон: =Цвета.Стилевые.ФонВторичный
            ШиринаВКолонках: Одинарная
            Содержимое:
                Тип: Группа
                Компоновка: Вертикальная
                Содержимое:
                    -
                        Тип: Надпись
                        Значение: "<Текст>"
                        МаксимальнаяВысотаВСтроках: 5
                        РастягиватьПоГоризонтали: Истина
                        Шрифт: =СтилевыеШрифты.ОченьКрупный
                    -
                        Тип: Надпись
                        Заголовок: <Должность>          # подпись под именем
                        Значение: "<Имя>"
                        Изображение: <Аватар.png>       # миниатюра аватара
        # ... повторить для каждого элемента
```

**Карусель из картинок (например, логотипы партнёров):**
```yaml
-
    Тип: Группа
    Компоновка: Карусель
    НастройкиКарусели:
        ИнтервалВоспроизведения: 3с
        ЦиклическаяНавигация: Истина
    Содержимое:
        -
            Тип: Картинка
            Изображение: <Файл1.png>
        -
            Тип: Картинка
            Изображение: <Файл2.png>
```

## P7: Бенто (плиточная сетка)

Плиточная сетка с карточками разного размера. Хорошо для категорий, преимуществ.

```yaml
-
    Тип: Группа
    Заголовок: <Подзаголовок секции>  # необязательно
    Компоновка: Бенто
    НастройкиКомпоновкиБенто:
        РазрешитьИзменениеПорядка: Ложь
    Содержимое:
        -
            Тип: СтандартнаяКарточка
            Заголовок: <Название>
            ШиринаВКолонках: Двойная    # увеличенная карточка
            ОбрабатыватьНажатие: Истина
            Содержимое: <Описание>      # необязательно
        -
            Тип: СтандартнаяКарточка
            Заголовок: <Название>
            ШиринаВКолонках: Одинарная  # стандартная карточка
            ОбрабатыватьНажатие: Истина
        # ... повторить
```

**Правила Бенто:**
- Смешивать `ШиринаВКолонках: Одинарная` и `Двойная` для разного визуального веса
- `ОбрабатыватьНажатие: Истина` без `ПриНажатии` — карточка кликабельна,
  обработчик добавить в .xbsl позже
- `НастройкиКомпоновкиБенто.РазрешитьИзменениеПорядка: Ложь` — фиксированный порядок

**Бенто для статичного списка (без кликов):**
```yaml
-
    Тип: Группа
    Компоновка: Бенто
    Содержимое:
        -
            Тип: СтандартнаяКарточка
            ШиринаВКолонках: Одинарная
            Заголовок: <Название>
            Содержимое: <Текст>
            РастягиватьПоВертикали: Истина
```

## Шаблон .xbsl файла

```xbsl
импорт <Подсистема1>
импорт <Подсистема2>

@Обработчик
метод ПослеСоздания()
    ЗаполнитьКарточки()
    
    // Подписаться на события обновления (если нужно):
    // <СобытиеОбновления>.ПодключитьОбработчик(() -> ЗаполнитьКарточки())
;

метод ЗаполнитьКарточки()
    ЗаполнитьКарточку<Имя1>()
    ЗаполнитьКарточку<Имя2>()
    // ОбновитьДиаграммы()
;

// P1: Заполнение KPI-счётчика
метод ЗаполнитьКарточку<Имя>()
    знч Данные = Получить<Имя>()
    Компоненты.НадписьЗначение1.Значение = Данные.<Поле1>.Представление()
    Компоненты.НадписьЗначение2.Значение = "${Данные.<Поле2>} руб."
;

// P2: Навигация по клику
метод Карточка<Раздел>ПриНажатии(Источник: Карточка, Событие: СобытиеПриНажатии)
    <Раздел>ФормаСписка.Открыть()
;

// P2: Открыть форму объекта в диалоге (кнопка «Создать»)
метод Создать<Объект>ПриНажатии(Команда: ОбычнаяКоманда)
    <Объект>ФормаОбъекта.Открыть(СпособОткрытия = СпособОткрытияФормы.ВДиалоговомОкне)
;

// P3: Заполнение таблицы
метод ЗаполнитьКарточку<Таблица>()
    Компоненты.Таблица<Имя>.Источник.Данные = Получить<Таблица>()
;

// P5: Обновление диаграммы
метод ОбновитьДиаграммы()
    Компоненты.<ИмяДиаграммы>.ОбновитьДиаграмму()
;

// Серверный метод (данные с сервера):
@НаСервере @ДоступноСКлиента
статический метод Получить<Имя>(): <ТипВозврата>
    возврат <Объект>.Получить<Имя>()
;
```

**Правила .xbsl:**
- НЕ используется `@КомпонентИнтерфейса` — методы на верхнем уровне файла
- `@Обработчик` — только для `ПослеСоздания`
- Обращение к именованным компонентам: `Компоненты.<ИмяКомпонента>.Значение`
- Серверные данные: `@НаСервере @ДоступноСКлиента статический метод`
- Импорт других модулей: `импорт <ИмяПодсистемы>` (в нижнем регистре, без пробела)

## Правила стилизации

**Цвета картинок-иконок (палитра стилевых цветов):**
```yaml
Цвет: =Цвета.Оранжевый
Цвет: =Цвета.Зеленый
Цвет: =Цвета.Голубой
Цвет: =Цвета.Красный
Цвет: =Цвета.Фиолетовый
```

**Фон карточек:**
```yaml
# Стилевые (предпочтительно):
Фон: =Цвета.Стилевые.ФонПервичный
Фон: =Цвета.Стилевые.ФонВторичный

# Кастомный цвет (для лендингов и баннеров):
Фон:
    Тип: АбсолютныйЦвет
    Значение: RGB(5955FF)
```

**Шрифты:**
```yaml
# Стилевые (предпочтительно, адаптируются к теме):
Шрифт: =СтилевыеШрифты.ЗаголовокОбычный
Шрифт: =СтилевыеШрифты.ЗаголовокКрупный
Шрифт: =СтилевыеШрифты.ОченьКрупный
Шрифт: =СтилевыеШрифты.Крупный

# Абсолютный (для кастомного дизайна):
ШрифтЗаголовка:
    Тип: АбсолютныйШрифт
    Полужирный: Истина
    Размер: 36
```

**Цвет текста (для кастомных цветных карточек):**
```yaml
ЦветЗаголовка:
    Тип: АбсолютныйЦвет
    Значение: RGB(FFFFFF)
ЦветСодержимого:
    Тип: АбсолютныйЦвет
    Значение: RGB(FFFFFF)
```

## Регистрация в навигационной панели

Форма **не появляется автоматически** — нужно зарегистрировать в навигации.

**Вариант 1: через ПанельНавигации.yaml** (стандартное приложение с разделами)
```yaml
# В файле ПанельНавигации.yaml — добавить первым элементом:
-
    Тип: НавигационнаяКоманда
    Изображение: <ИконкаГлавная.svg>
    Представление: Главная
    ТипФормы: <ИмяФормы>
```

**Вариант 2: через Приложение*.yaml** (клиентское приложение)
```yaml
# В КомандныйИнтерфейсПанелиНавигации добавить:
КомандныйИнтерфейсПанелиНавигации:
    Тип: ФрагментКомандногоИнтерфейса
    Элементы:
        -
            Тип: НавигационнаяКоманда
            Представление: <Название>
            ТипФормы: <ИмяФормы>
```

```xbsl
импорт Заказы

@Обработчик
метод ПослеСоздания()
    ЗаполнитьКарточкуЗаказов()
;

метод ЗаполнитьКарточкуЗаказов()
    знч КоличествоЗаказов = ПолучитьКоличествоЗаказов()

метод КарточкаКлиентовПриНажатии(Источник: Карточка, Событие: СобытиеПриНажатии)

метод СоздатьЗаказПриНажатии(Команда: ОбычнаяКоманда)
    ЗаказыФормаОбъекта.Открыть(СпособОткрытия = СпособОткрытияФормы.ВДиалоговомОкне)
```
