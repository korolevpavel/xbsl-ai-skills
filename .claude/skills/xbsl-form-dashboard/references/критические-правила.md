# Критические правила (проверено на реальных проектах)

## 1. Статические строки в `Значение:` требуют кавычек

```yaml
# ✅ Правильно:
Значение: "Заказов"
Значение: "В процессе"

# ❌ Ошибка компиляции:
Значение: Заказов
```

Только статические строковые литералы. Динамические выражения (`=...`) и локализованные строки (`$...`) — без кавычек.

---

## 2. SVG-иконки — только платформенные или из папки Ресурсы/

```yaml
# ✅ Проверенные платформенные иконки:
Изображение: Плюс.svg
Изображение: Задачи.svg
Изображение: Лупа.svg
Изображение: Аккаунт.svg

# ✅ Иконки из папки Ресурсы/ подсистемы:
Изображение: МояИконка.svg   # файл в <Подсистема>/Ресурсы/МояИконка.svg

# ❌ Ошибка "Неизвестный ресурс":
Изображение: Документ.svg
Изображение: Склад.svg
Изображение: Заказы.svg
```

---

## 3. Видимость объектов из других подсистем

```yaml
# ✅ Доступно из любой подсистемы при импорте:
ОбластьВидимости: ВПроекте

# ❌ Ошибка "invisible due to visibility modifier":
ОбластьВидимости: ВПодсистеме
```

Все формы, объекты, структуры, на которые ссылается ГлавнаяСтраница из другой подсистемы,
должны иметь `ОбластьВидимости: ВПроекте`.

**Перед генерацией обработчиков навигации** — проверить видимость целевых форм:

```bash
grep "ОбластьВидимости" <путь>/<Подсистема>/<ФормаСписка>.yaml
```

Если `ВПодсистеме` — исправить на `ВПроекте` до деплоя, иначе ошибка компиляции:
`Variable "ХххФормаСписка" is not defined`

---

## 4. Круговые зависимости подсистем → выделить подсистему Дашборд

Если подсистема `Склад` использует (`Использование: [Основное]`) — то `Основное` не может
импортировать `Склад`. Это круговая зависимость.

**Решение:** создать верхнеуровневую подсистему `Дашборд`:

```yaml
# Дашборд/Подсистема.yaml
Использование:
    - Основное
    - Склад       # или любая другая, с которой конфликт
    - Закупки
Интерфейс:
    ВключатьВАвтоИнтерфейс: Ложь
```

Перенести в `Дашборд/`: `ГлавнаяСтраница.yaml`, `ГлавнаяСтраница.xbsl`, `Приложение.yaml`,
вспомогательные структуры (СтрокаСотрудника и т.п.).

---

## 5. Структура для типизированных таблиц и списков

`Структура` (тип данных для `ИсточникДанныхМассив<T>`) требует:

```yaml
ВидЭлемента: Структура
Ид: <UUID>
Имя: СтрокаМоегоОбъекта
ОбластьВидимости: ВПроекте
Окружение: КлиентИСервер       # ← обязательно, иначе "Environment is required"
Поля:
    -
        Имя: Поле1             # ← без Ид внутри полей — вызывает ошибку
        Тип: Строка
    -
        Имя: Поле2
        Тип: Число
```

Ошибки при нарушении:
- Нет `Окружение:` → `Environment is required`
- Есть `Ид:` внутри поля → `Неизвестное свойство "Ид"`

---

## 6. Для `Тип: Форма` нет авто-команды `.Открыть`

В отличие от Справочников и Документов, форма типа `Форма` не генерирует авто-команду.

```yaml
# ❌ Ошибка компиляции:
- =ГлавнаяСтраница.Открыть

# ✅ Правильно — только через НавигационнаяКоманда:
-
    Тип: НавигационнаяКоманда
    ТипФормы: ГлавнаяСтраница
    Представление: Главная
    Изображение: Плюс.svg
```

Без `КомпонентИнтерфейса` с `Тип: СтандартноеКлиентскоеПриложениеСРазделами`
платформа строит авто-навигацию и форму с `ВключатьВАвтоИнтерфейс: Ложь` не показывает.
