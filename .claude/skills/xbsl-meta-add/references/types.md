# Типы данных реквизитов

| Тип | Пример использования | Доп. свойства |
|-----|---------------------|---------------|
| `Строка` | ФИО, телефон, адрес, текст | `МаксимальнаяДлина: 200` |
| `Число` | цена, количество, возраст | `ДлинаЦелойЧасти`, `ДлинаДробнойЧасти`, `МинимальноеЗначение` |
| `Булево` | активен, опубликован, флажок | — |
| `Дата` | дата рождения, дата приёма | — |
| `ДатаВремя` | дата и время события | — |
| `Момент` | временная метка с таймзоной | — |
| `ДвоичныйОбъект.Ссылка?` | фото, прикреплённый файл | — |
| `<Справочник>.Ссылка?` | ссылка на другой справочник | `НезаполненноеЗначение: ЗапретитьВсегда` если обязательно; если объект из другой подсистемы — он должен иметь `ОбластьВидимости: ВПроекте` |
| `<Документ>.Ссылка?` | ссылка на документ | аналогично; если из другой подсистемы — объект должен иметь `ОбластьВидимости: ВПроекте` |
| `<Перечисление>?` | статус, вид, роль (nullable — по умолчанию) | `ИспользоватьВПолнотекстовомПоиске: Ложь` |
| `<Перечисление>` | статус с обязательным значением | требует `ЗначениеПоУмолчанию: <Перечисление>.<Элемент>` |
| `Пользователи.Ссылка?` | системный пользователь | — |
| `Массив<<Тип>.Ссылка>` | список ссылок одного типа | — |
| `Массив<<ТипА>.Ссылка\|<ТипБ>.Ссылка>` | список ссылок разных типов (union) | — |

## Параметры числовых полей

```yaml
Тип: Число
ДлинаЦелойЧасти: 10
ДлинаДробнойЧасти: 2
МинимальноеЗначение: 0
МаксимальноеЗначение: 9999999
ЗначениеПоУмолчанию: 0
```

## Параметры строковых полей

```yaml
Тип: Строка
МаксимальнаяДлина: 200      # для обычных строк
# или
Длина: 3                     # фиксированная длина (для кодов)
Многострочная: Истина        # для текстовых областей (описание, примечание)
```

## Флаги полей

- `НезаполненноеЗначение: ЗапретитьВсегда` — поле обязательно к заполнению (пользовательские поля; на `Наименование`, `Код`, `Дата`, `Номер` не ставится)
- `ИспользоватьВПолнотекстовомПоиске: Ложь` — для ссылок, перечислений, бинарных данных
- Поле с типом-перечислением **всегда** должно быть либо `<Перечисление>?` (nullable), либо иметь `ЗначениеПоУмолчанию`. Иначе проект не соберётся.
- `ЗаполнятьПриКопировании: Ложь` — для уникальных значений (пользователь, дата создания)
- `Уникальность: Истина` — значение должно быть уникальным в рамках объекта
