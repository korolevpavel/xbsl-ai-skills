# Спецификация: ГлобальноеКлиентскоеСобытие

## Дефолты

- **Имя** — PascalCase, глагол в прошедшем времени (`ЗаписанаЗадача`, `УдалёнКонтрагент`, `ИзменёнСтатус`)
- **Видимость** — `ВПроекте` (события обычно нужны в нескольких подсистемах)

## UUID

Нужен: 1 (объект). Параметры UUID не требуют.

## Структура YAML

### Минимальная (событие без параметров):

```yaml
ВидЭлемента: ГлобальноеКлиентскоеСобытие
Ид: <UUID>
Имя: <ИмяСобытия>
ОбластьВидимости: ВПроекте
```

### С параметрами:

```yaml
ВидЭлемента: ГлобальноеКлиентскоеСобытие
Ид: <UUID>
Имя: <ИмяСобытия>
ОбластьВидимости: ВПроекте
Параметры:
    - Имя: <ИмяПараметра>
      Тип: <Тип>?              # чаще опциональный
    - Имя: <ИмяПараметра>
      Тип: <Справочник>.Ссылка?
```

## Правила

- `Параметры` — данные, передаваемые подписчикам события; типы аналогичны реквизитам
- Имя в прошедшем времени отражает факт свершившегося действия
- `ОбластьВидимости: ВПроекте` — если на событие подписываются разные подсистемы

## XBSL паттерны

### Генерация события (в обработчике записи документа/справочника):

```xbsl
@Обработчик
метод ПослеЗаписи()
    Приложение.ВызватьГлобальноеКлиентскоеСобытие(
        &<ИмяСобытия>,
        <Параметр1>,
        <Параметр2>
    )
;
```

### Подписка на событие (в КомпонентеИнтерфейса):

```xbsl
@Обработчик
метод ПриОткрытии()
    Приложение.ПодписатьсяНаГлобальноеКлиентскоеСобытие(
        &<ИмяСобытия>,
        &ОбработатьСобытие
    )
;

метод ОбработатьСобытие(<Параметр1>: <Тип>?)
    // реакция на событие
    ОбновитьСписок()
;
```
