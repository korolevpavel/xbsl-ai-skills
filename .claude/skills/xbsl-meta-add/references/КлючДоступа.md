# Спецификация: КлючДоступа

## Дефолты

- **Имя** — PascalCase, описывает роль/контекст (`КлючДоступаМенеджера`, `КлючДоступаАдминистратора`, `КлючДоступаСотрудника`)
- **Видимость** — `ВПроекте` (ключи доступа обычно используются в нескольких местах)

## UUID

Нужен: 1 (объект) + по 1 на каждый параметр (если есть).

## Структура YAML

### Минимальная (без параметров):

```yaml
ВидЭлемента: КлючДоступа
Ид: <UUID>
Имя: <ИмяКлюча>
ОбластьВидимости: ВПроекте
```

### С параметрами:

```yaml
ВидЭлемента: КлючДоступа
Ид: <UUID>
Имя: <ИмяКлюча>
ОбластьВидимости: ВПроекте
Импорт:                          # необязательно
    - <ИмяПодсистемы>
Параметры:
    - Ид: <UUID>
      Имя: Владелец
      Тип: <Справочник>.Ссылка?  # кому принадлежит доступ
    - Ид: <UUID>
      Имя: Роль
      Тип: <Перечисление>        # уровень или вид доступа
```

## Правила

- Ключ доступа — именованный маркер, по которому система определяет права
- `Параметры` — уточняют контекст доступа (чей доступ, к какому объекту, с какой ролью)
- Ключи без параметров — для глобальных ролей (`КлючДоступаАдминистратора`)
- Ключи с параметрами — для доступа к конкретным объектам (`КлючДоступаМенеджера` с `Владелец`)

## Использование в коде

```xbsl
// В КонтрольДоступа объекта — вычисление разрешений
@Обработчик
метод ВычислитьРазрешенияДоступа(): ЧитаемаяКоллекция<РазрешениеДоступа>
    возврат [
        новый РазрешениеДоступа(
            [новый КлючДоступаМенеджера.Объект(Владелец = этот.Ответственный)],
            [Сущность.Право.Чтение, Сущность.Право.Изменение]
        ),
        новый РазрешениеДоступа(
            [новый КлючДоступаАдминистратора.Объект()],
            [Сущность.Право.Чтение, Сущность.Право.Создание, Сущность.Право.Изменение, Сущность.Право.Удаление]
        )
    ]
;

// Проверка доступа в коде
если !Доступ.ПроверитьКлюч(новый КлючДоступаМенеджера.Объект(Владелец = Сотрудник))
    ВызватьИсключение "Недостаточно прав"
;
```
