# Спецификация: ЛокализованныеСтроки

## Дефолты

- **Имя** — всегда `ЛокализованныеСтроки` (по одному объекту на подсистему)
- **Видимость** — `ВПодсистеме`

## UUID

Нужен: 1 (объект). Строки UUID не требуют.

## Структура YAML

### Минимальная:

```yaml
ВидЭлемента: ЛокализованныеСтроки
Ид: <UUID>
Имя: ЛокализованныеСтроки
ОбластьВидимости: ВПодсистеме
Строки:
    <КлючСтроки>: <Текст строки>
    <КлючСтроки>: <Текст строки>
```

### Полная (со строками и шаблонами):

```yaml
ВидЭлемента: ЛокализованныеСтроки
Ид: <UUID>
Имя: ЛокализованныеСтроки
ОбластьВидимости: ВПодсистеме
Строки:
    НовыйЗаказ: Новый заказ
    ЗаказСохранён: Заказ успешно сохранён
    ОшибкаСохранения: Не удалось сохранить заказ
Шаблоны:                          # необязательно — строки с подстановкой параметров
    СозданЗаказНомер: "Создан заказ \"$0\""
    ОшибкаПоляЗначение: "Поле \"$0\" содержит недопустимое значение: \"$1\""
```

## Правила

- Ключи строк — PascalCase: `НовыйЗаказ`, `ОшибкаСохранения`
- `Строки` — статические тексты без параметров
- `Шаблоны` — тексты с подстановками `$0`, `$1`, `$N` (нумерация с нуля)
- Один файл `ЛокализованныеСтроки.yaml` на подсистему — собирает все строки подсистемы

## Использование в коде

```xbsl
// Ссылка на строку в YAML-поле Представление
Представление: $ЛокализованныеСтроки.НовыйЗаказ

// Ссылка в XBSL коде
знч Текст = $ЛокализованныеСтроки.ЗаказСохранён

// Шаблон с подстановкой
знч Сообщение = ФорматировщикСтрок.ФоматСтроки(
    $ЛокализованныеСтроки.СозданЗаказНомер,
    НомерЗаказа
)
```
