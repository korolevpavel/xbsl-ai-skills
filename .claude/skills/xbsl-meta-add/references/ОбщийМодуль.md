# Спецификация: ОбщийМодуль

## Дефолты

- **Имя** — PascalCase, отражает назначение (`ПользователиКлиентИСервер`, `УтилитыСервиса`, `СериализацияJson`)
- **Видимость** — `ВПодсистеме`
- **Окружение** — `КлиентИСервер` (если нужен и на сервере, и на клиенте)

## UUID

Нужен: 1 (объект).

## Структура YAML

### Минимальная:

```yaml
ВидЭлемента: ОбщийМодуль
Ид: <UUID>
Имя: <ИмяМодуля>
ОбластьВидимости: ВПодсистеме
Окружение: КлиентИСервер
```

### Полная:

```yaml
ВидЭлемента: ОбщийМодуль
Ид: <UUID>
Имя: <ИмяМодуля>
ОбластьВидимости: ВПроекте    # или ВПодсистеме
Окружение: КлиентИСервер      # или Сервер, Клиент
Импорт:
    - <ИмяПодсистемы>
```

## Правила

- `Окружение: Сервер` — методы доступны только на сервере (работа с БД, внешние вызовы)
- `Окружение: Клиент` — методы доступны только на клиенте (UI, уведомления)
- `Окружение: КлиентИСервер` — методы доступны в обоих контекстах; логика не должна зависеть от контекста выполнения
- `Импорт` — список подсистем, чьи объекты используются в методах модуля

## XBSL паттерны

### Серверный метод с доступом с клиента:

```xbsl
@ВПроекте
@НаСервере
@ДоступноСКлиента
метод ПолучитьТекущегоСотрудника(): Сотрудники.Ссылка?
    // реализация
;
```

### Утилитарный метод (КлиентИСервер):

```xbsl
@ВПодсистеме
метод ПроверитьЗаполненность(Значение: Строка?): Булево
    возврат Значение != Неопределено && Значение != ""
;
```

### Метод, реализующий контракт:

```xbsl
@ВПроекте
@Реализация
метод ОписаниеПримера(): Пример.ОписаниеПримера
    возврат новый Пример.ОписаниеПримера(
        Название = "Название примера",
        КраткоеОписание = "Описание"
    )
;
```
