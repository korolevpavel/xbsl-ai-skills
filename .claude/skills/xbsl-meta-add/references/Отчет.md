# Спецификация: Отчет

## Дефолты

- **Имя** — PascalCase, существительное (`ОстаткиТоваров`, `АнализПродаж`, `ОтчетПоДолжникам`)
- **Видимость** — `ВПодсистеме`
- **ВидИсточникаДанных** — `Таблица` (default) для обычных таблиц (Справочник, Документ, РегистрСведений). Для виртуальных таблиц РН (`.Остатки`, `.Обороты`) **обязательно** используй `Запрос` + создавай `.xbql` файл — `Таблица` не поддерживает агрегацию для виртуальных таблиц
- **ВключатьВАвтоИнтерфейс** — `Истина` по умолчанию
- **Макет** — генерируется AI на основе полей источника данных (см. раздел «Макет»)
- **ПараметрыЗапроса** — добавлять только если используется `ВидИсточникаДанных: Запрос` и пользователь явно указал параметры

## UUID

Нужен: **1** (только на объект, формат со дефисами) + по **1 на каждое поле `Макет.Поля`** (hex без дефисов). Вызывай `xbsl-uuid` дважды: для объекта — обычный формат, для полей Макета — hex-формат. `ПараметрыЗапроса` — без UUID.

## Структура YAML

### Вариант: Таблица (источник данных — таблица объекта)

```yaml
ВидЭлемента: Отчет
Ид: <UUID>
Имя: <ИмяОтчета>
ОбластьВидимости: ВПодсистеме
Представление: <Заголовок для UI>
Импорт:
    - <ПодсистемаИсточника>   # если ИсточникДанных из другой подсистемы
ВидИсточникаДанных: Таблица
ИсточникДанных: <ИмяТаблицы>      # напр. Сотрудники или Остатки.Остатки
Макет:
    ВидОтображения: СводнаяТаблица
    Поля:
        -
            Вид: Измерение
            ВизуальныеРоли:
                - Строки
            Выражение: <ИмяИзмерения>
            Ид: <UUID32hex>
            Использовать: Истина
        -
            Вид: Мера
            ВизуальныеРоли:
                - ЗначенияКолонок
            Выражение: СУММА(<ИмяРесурса>)
            Ид: <UUID32hex>
            Использовать: Истина
            Представление: <ЗаголовокКолонки>
```

### Вариант: Запрос (произвольный XBSL-запрос)

```yaml
ВидЭлемента: Отчет
Ид: <UUID>
Имя: <ИмяОтчета>
ОбластьВидимости: ВПодсистеме
Представление: <Заголовок для UI>
ВидИсточникаДанных: Запрос
Интерфейс:
    ВключатьВАвтоИнтерфейс: Истина
    Форма: <ИмяОтчета>ФормаОтчета  # только если пользователь просит создать форму
ПараметрыЗапроса:                  # необязательно — только если есть параметры в запросе
    -
        Имя: <ИмяПараметра>
        Тип: <Тип>
        ЗначениеПоУмолчанию: <Значение>   # необязательно
```

## Макет

Всегда генерируй `Макет` для `ВидИсточникаДанных: Таблица`. По умолчанию используй `ВидОтображения: СводнаяТаблица`.

### Правила полей

| Тип поля источника | `Вид` в Макет | `ВизуальныеРоли` | `Выражение` |
|---|---|---|---|
| Первое измерение (строки) | `Измерение` | `- Строки` | Имя поля как есть |
| Второе измерение (колонки) | `Измерение` | `- Колонки` (+ `- Итог` если нужен итог) | Имя поля как есть |
| Ресурс обычной таблицы | `Мера` | `- ЗначенияКолонок` | `СУММА(<ИмяРесурса>)` |
| Ресурс виртуальной таблицы `.Остатки` | `Мера` | `- ЗначенияКолонок` | `СУММА(<ИмяРесурсаОстаток>)` — с суффиксом `Остаток` |
| Ресурс виртуальной таблицы `.Обороты` | `Мера` | `- ЗначенияКолонок` | `СУММА(<ИмяРесурсаTurnover>)` — с суффиксом `Turnover` |

> **Важно:** `ЗначенияКолонок` у Меры работает только при наличии хотя бы одного измерения с ролью `Колонки`. Без него ресурс не отображается. Если измерений несколько — одно помести в `Строки`, другое в `Колонки`.

### UUID полей Макета

Каждое поле `Макет.Поля` требует `Ид` в формате **32-символьного hex без дефисов** (UUID v4 без дефисов).
Генерируй через `python3 -c "import uuid; print(uuid.uuid4().hex)"` — по одному на каждое поле.

### Пример: отчёт по виртуальной таблице РН Остатки

Источник `РегистрОстатков.Остатки`, измерения `Номенклатура` (строки) и `Склад` (колонки), ресурс `Количество`:

```yaml
Макет:
    ВидОтображения: СводнаяТаблица
    Поля:
        -
            Вид: Измерение
            ВизуальныеРоли:
                - Строки
            Выражение: Номенклатура
            Ид: <UUID32hex>
            Использовать: Истина
        -
            Вид: Измерение
            ВизуальныеРоли:
                - Колонки
                - Итог
            Выражение: Склад
            Ид: <UUID32hex>
            Использовать: Истина
        -
            Вид: Мера
            ВизуальныеРоли:
                - ЗначенияКолонок
            Выражение: СУММА(КоличествоОстаток)
            Ид: <UUID32hex>
            Использовать: Истина
            Представление: Количество
```

## Правила

- `Макет` — **всегда генерируй** для `ВидИсточникаДанных: Таблица`. Для `Запрос` — только если пользователь явно перечислил поля
- `ИсточникДанных` — только для `ВидИсточникаДанных: Таблица`. Для виртуальных таблиц РН (`.Остатки`, `.Обороты`) **не используй `Таблица`** — переключись на `Запрос` + `.xbql` файл с явным ВЫБРАТЬ-запросом
- `.xbql` файл — при `ВидИсточникаДанных: Запрос` создай `{ИмяОтчета}.xbql` рядом с YAML. Для виртуальной таблицы РН: `ВЫБРАТЬ <Рег>Остатки.Измерение1, <Рег>Остатки.<Ресурс>Остаток ИЗ <Рег>.Остатки() КАК <Рег>Остатки`
- **`ИсточникДанных`** — при `ВидИсточникаДанных: Запрос` это поле **запрещено** (не добавляй его вообще, даже пустым)
- `Импорт` — если `ИсточникДанных` ссылается на объект из **другой** подсистемы, добавь блок `Импорт` с именем этой подсистемы. Без него платформа выдаёт ошибку `Namespace "..." is not imported`
- `Подсистема.yaml` — если `ИсточникДанных` из другой подсистемы, в `Подсистема.yaml` подсистемы отчёта добавь `Использование: - <ПодсистемаИсточника>`. Без этого — ошибка `Subsystem "..." is not specified as used`
- `ОбластьВидимости` объекта-источника — должна быть `ВПроекте`, иначе ошибка `Table "..." is invisible due to visibility modifier @ВПодсистеме`. Если источник `ВПодсистеме` — измени на `ВПроекте`
- Если `ВидИсточникаДанных: Таблица` — поле `ИсточникДанных` обязательно; `ПараметрыЗапроса` не нужны
- Если `ВидИсточникаДанных: Запрос` — поле `ИсточникДанных` не указывается; `ПараметрыЗапроса` — только по явному запросу пользователя
- Типы в `ПараметрыЗапроса` берутся из `references/types.md` (xbsl-meta-add)
- Блок `Интерфейс.Форма` — добавляй только если пользователь просит создать форму одновременно
- Блок `Интерфейс` можно опустить полностью если `ВключатьВАвтоИнтерфейс: Истина` и форма не задаётся
