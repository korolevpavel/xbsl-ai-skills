# Спецификация: РегистрНакопления

## Дефолты

- **Имя** — PascalCase, существительное во множественном числе (`Продажи`, `ОстаткиНоменклатуры`)
- **Видимость** — `ВПодсистеме`
- **ВидРегистра** — уточни у пользователя: `Обороты` или `Остатки`

## UUID

Нужны: 1 (объект) + по 1 на каждое Измерение + по 1 на каждый Ресурс + по 1 на каждый Реквизит.

## Структура YAML

### Минимальная:

```yaml
ВидЭлемента: РегистрНакопления
Ид: <UUID>
Имя: <ИмяРегистра>
ОбластьВидимости: ВПодсистеме
ВидРегистра: Обороты             # или Остатки
Измерения:
    - Ид: <UUID>
      Имя: <ИмяИзмерения>
      Тип: <Справочник>.Ссылка?
Ресурсы:
    - Ид: <UUID>
      Имя: <ИмяРесурса>
      Тип: Число
      ДлинаЦелойЧасти: 15
      ДлинаДробнойЧасти: 2
```

### Полная:

```yaml
ВидЭлемента: РегистрНакопления
Ид: <UUID>
Имя: <ИмяРегистра>
ОбластьВидимости: ВПодсистеме
ВидРегистра: Остатки             # или Обороты
Импорт:
    - Основное
Интерфейс:
    ВключатьВАвтоИнтерфейс: Ложь
    Список:
        Форма: <ИмяФормыСписка>
КонтрольДоступа:
    Разрешения:
        ПоУмолчанию: РазрешенияВычисляютсяДляКаждогоОбъекта
    РасчетРазрешенийПо:
        - <ИмяИзмерения>
Измерения:
    - Ид: <UUID>
      Имя: <ИмяИзмерения>
      Тип: <Справочник>.Ссылка?
      НезаполненноеЗначение: ЗапретитьВсегда
Ресурсы:
    - Ид: <UUID>
      Имя: <ИмяРесурса>
      Тип: Число
      ДлинаЦелойЧасти: 15
      ДлинаДробнойЧасти: 2
      МинимальноеЗначение: 0
Реквизиты:                       # необязательно — часто содержит ссылку на регистратор
    - Ид: <UUID>
      Имя: Регистратор
      Тип: <Документ>.Ссылка?    # или union: <ДокА>.Ссылка|<ДокБ>.Ссылка|?
```

## Правила

- `ВидРегистра: Обороты` — регистрирует приход и расход; данные накапливаются, не перезаписываются; виртуальная таблица `СрезОборотов`
- `ВидРегистра: Остатки` — хранит остатки; виртуальная таблица `СрезОстатков` и `СрезПоследних`
- `Измерения` — по каким объектам накапливаются данные (номенклатура, склад, контрагент)
- `Ресурсы` — что накапливается (количество, сумма)
- `Реквизиты` — дополнительные поля движения (часто `Регистратор` — ссылка на документ)

## XBSL паттерны

### Запрос остатков (ВидРегистра: Остатки):

```xbsl
@ВПодсистеме
метод ПолучитьОстаток(Номенклатура: <Справочник>.Ссылка): Число
    знч Запрос = Запрос{
        ВЫБРАТЬ
            Таблица.Количество КАК Количество
        ИЗ
            <ИмяРегистра>.СрезОстатков() КАК Таблица
        ГДЕ
            Таблица.<Измерение> == %Номенклатура
    }
    возврат Запрос.Выполнить().ПервыйИлиНеопределено()?.Количество ?? 0
;
```

### Запрос оборотов (ВидРегистра: Обороты):

```xbsl
знч Запрос = Запрос{
    ВЫБРАТЬ
        Таблица.<Измерение>,
        СУММА(Таблица.<Ресурс>) КАК Итого
    ИЗ
        <ИмяРегистра>.СрезОборотов(%НачалоПериода, %КонейПериода) КАК Таблица
    СГРУППИРОВАТЬ ПО
        Таблица.<Измерение>
}
```
