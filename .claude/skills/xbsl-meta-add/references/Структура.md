# Спецификация: Структура

## Дефолты

- **Имя** — PascalCase, единственное число (`ДанныеКонтрагента`, `КурсВалюты`, `ПараметрыФильтра`)
- **Видимость** — `ВПодсистеме`
- **Окружение** — `КлиентИСервер` (если не нужна специфика одного контекста)

## UUID

Нужен: 1 (объект). Поля структуры UUID не требуют.

## Структура YAML

### Минимальная:

```yaml
ВидЭлемента: Структура
Ид: <UUID>
Имя: <ИмяСтруктуры>
ОбластьВидимости: ВПодсистеме
Поля:
    - Имя: <ИмяПоля>
      Тип: <Тип>
```

### Полная:

```yaml
ВидЭлемента: Структура
Ид: <UUID>
Имя: <ИмяСтруктуры>
ОбластьВидимости: ВПроекте     # или ВПодсистеме
Окружение: КлиентИСервер       # или Сервер, Клиент
Импорт:
    - <ИмяПодсистемы>
Поля:
    - Имя: <ИмяПоля>
      Тип: <Тип>
      Обязательное: Истина      # необязательно — поле должно быть заполнено
    - Имя: <ИмяПоля>
      Тип: Строка?
      Аннотации:                # необязательно — для JSON-маппинга и другого
          - Тип: JsonСвойство
            Имя: <json_key>     # имя ключа в JSON (если отличается от Имя)
```

## Правила

- Структура — это DTO / value object: хранит данные без логики
- `Обязательное: Истина` — поле должно быть передано при создании экземпляра
- `Тип: <Тип>?` (с `?`) — поле опциональное, может быть `Неопределено`
- Аннотация `JsonСвойство` — задаёт имя ключа при сериализации/десериализации JSON
- Структуры обычно не содержат XBSL-методов

## Использование в коде

```xbsl
// Создание экземпляра
знч Данные = новый ДанныеКонтрагента(
    Наименование = "ООО Ромашка",
    ИНН = "1234567890"
)

// Десериализация из JSON
знч Данные: ДанныеКонтрагента = СериализацияJson.ПрочитатьОбъект(
    Запрос.Тело,
    Тип<ДанныеКонтрагента>
)

// Сериализация в JSON
знч Json = СериализацияJson.ЗаписатьОбъект(Данные)
```
