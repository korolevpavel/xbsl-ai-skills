# Спецификация: Табличные части

Табличные части используются в `Справочник` и `Документ` для хранения списка строк, связанных с объектом.

## UUID

Каждая ТЧ требует отдельного UUID. Каждый реквизит ТЧ тоже требует UUID. Системных полей в ТЧ нет.

**Формула подсчёта:** +1 за ТЧ + по 1 за каждый реквизит ТЧ.

## Структура YAML

```yaml
ТабличныеЧасти:
    -
        Ид: <UUID>
        Имя: <ИмяТЧ>
        Реквизиты:
            -
                Ид: <UUID>
                Имя: <ИмяКолонки>
                Тип: <Тип>
            -
                Ид: <UUID>
                Имя: <ИмяКолонки2>
                Тип: <Тип>
```

## Правила

- `ТабличныеЧасти` располагается после `Реквизиты` объекта.
- Имя ТЧ — PascalCase, существительное во множественном числе (`Товары`, `Контакты`, `Услуги`).
- Реквизиты ТЧ подчиняются тем же правилам типов, что и реквизиты объекта:
  - Ссылочные поля и перечисления: `ИспользоватьВПолнотекстовомПоиске: Ложь`
  - Обязательные поля: `НезаполненноеЗначение: ЗапретитьВсегда`
  - Числа: указывать `ДлинаЦелойЧасти`, `ДлинаДробнойЧасти`, `МинимальноеЗначение`
  - Строки: указывать `МаксимальнаяДлина`

## Пример: ТЧ Товары

```yaml
ТабличныеЧасти:
    -
        Ид: <UUID>
        Имя: Товары
        Реквизиты:
            -
                Ид: <UUID>
                Имя: Номенклатура
                Тип: Номенклатура.Ссылка?
                НезаполненноеЗначение: ЗапретитьВсегда
                ИспользоватьВПолнотекстовомПоиске: Ложь
            -
                Ид: <UUID>
                Имя: Количество
                Тип: Число
                ДлинаЦелойЧасти: 15
                ДлинаДробнойЧасти: 3
                МинимальноеЗначение: 0
            -
                Ид: <UUID>
                Имя: Цена
                Тип: Число
                ДлинаЦелойЧасти: 15
                ДлинаДробнойЧасти: 2
                МинимальноеЗначение: 0
            -
                Ид: <UUID>
                Имя: Сумма
                Тип: Число
                ДлинаЦелойЧасти: 15
                ДлинаДробнойЧасти: 2
                МинимальноеЗначение: 0
```

## Пример: ТЧ с контактами (справочник)

```yaml
ТабличныеЧасти:
    -
        Ид: <UUID>
        Имя: Контакты
        Реквизиты:
            -
                Ид: <UUID>
                Имя: ТипАдреса
                Тип: ТипыКонтактнойИнформации?
                ИспользоватьВПолнотекстовомПоиске: Ложь
            -
                Ид: <UUID>
                Имя: Значение
                Тип: Строка
                МаксимальнаяДлина: 200
```
