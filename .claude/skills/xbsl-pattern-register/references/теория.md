# Теория: регистр накопления в XBSL

Этот файл объясняет **почему** работают те или иные правила.
Читай, если нужно разобраться в механике, а не просто написать код.

---

## ВидЗаписи — только для регистров Остатки

Регистр с `ВидРегистра: Остатки` хранит текущий итог: платформа суммирует Приход и вычитает Расход.
Поэтому каждой записи нужно знать её "направление" — это и есть `ВидЗаписи`.

Регистр с `ВидРегистра: Обороты` просто накапливает сумму операций — понятия "направления" нет.
Указывать `ВидЗаписи` для него — синтаксическая ошибка.

---

## Фильтр по Регистратору

`НаборЗаписей.Фильтр.Установить(Регистратор = Ссылка)` сообщает платформе, что этот НаборЗаписей
отвечает именно за записи данного документа в данном регистре.

При вызове `Записать()` платформа:
1. Удаляет из регистра все старые записи этого документа (по фильтру).
2. Записывает новые записи из набора.

Если не установить фильтр — старые движения не удалятся, при повторном проведении записи задвоятся.

Пустой набор с установленным фильтром = удаление всех движений (паттерн отмены проведения).

## Суффиксы в виртуальной таблице `.Остатки`

Виртуальная таблица `<ИмяРегистра>.Остатки` — это не физическая таблица, а вычисляемый вид.
Платформа переименовывает поля ресурсов, добавляя суффикс `Остаток`, чтобы отличить их от измерений.

| Тип поля | Имя в YAML | Имя в запросе к `.Остатки` |
|---|---|---|
| Измерение | `Склад` | `Остатки.Склад` |
| Измерение | `Номенклатура` | `Остатки.Номенклатура` |
| Ресурс | `Количество` | `Остатки.КоличествоОстаток` |
| Ресурс | `Сумма` | `Остатки.СуммаОстаток` |

В `ДобавитьЗапись` суффиксов нет — там используются оригинальные имена из YAML.

## Контроль остатков: постфактум, через исключение

Паттерн B работает так:
1. Записываем движения в регистр (`НаборЗаписей.Записать()`).
2. Запрашиваем виртуальную таблицу `.Остатки` — ищем строки с отрицательным ресурсом.
3. Если нашли — выбрасываем исключение `ИсключениеНедопустимоеСостояние`.
4. Платформа откатывает транзакцию: движения не сохраняются, документ не проводится.

Это проще и надёжнее, чем превентивная проверка ДО записи:
- Не нужно блокировать строки регистра перед проверкой.
- Платформа сама гарантирует атомарность через транзакцию.

## РегистрСведений: механика

### Нет встроенного Регистратора в РС

В РС **нет** встроенного механизма Регистратора, как в РН. В РН `Регистратор` — системное поле, входящее в физический ключ записи; именно поэтому `НаборЗаписей.Фильтр.Установить(Регистратор = Ссылка)` работает в РН.

В РС физический ключ = `Измерения + Период`. Платформа не поддерживает привязку записей РС к документу-источнику.

#### Почему фильтр РС не принимает реквизиты

В **РН** `Регистратор` — встроенное системное поле, входящее в **физический ключ записи** (индексируется платформой). `НаборЗаписей.Фильтр` работает именно по ключевым полям — поэтому фильтр по Регистратору возможен.

В **РС** физический ключ = `Измерения + Период`. Реквизиты — хранимые данные, **не входящие в ключ**. Платформа не может выбрать «набор для записи» по реквизиту без полного сканирования — поэтому фильтр по реквизитам недоступен.

---

### Фильтр по измерениям

У РС нет встроенного регистратора — фильтр НаборЗаписей работает только по **измерениям**.
Записи не привязаны к документу через платформенный механизм.

```xbsl
// Фильтр по конкретному измерению — замещает только записи этого товара
НаборЗаписей.Фильтр.Установить(Товар = ТоварСсылка)

// Пустой фильтр — все записи регистра (используй осторожно!)
НаборЗаписей.Фильтр.Установить()
```

Фильтр **обязан** быть инициализирован до вызова `ДобавитьЗапись()`, `Прочитать()` и `Записать()`.

---

### Параметр Замещать в Записать()

`Записать(Замещать: Булево = Истина)` — управляет поведением при пересечении ключей:

| Замещать | Поведение |
|---|---|
| `Истина` (по умолчанию) | Все записи в фильтре удаляются, затем записываются новые |
| `Ложь` | Новые записи добавляются; если запись с таким ключом уже есть — она **не трогается** |

Пустой набор + `Записать()` (любой `Замещать`) = удаление всех записей в фильтре.

---

### Периодичность и поле Период

`Периодичность` в YAML определяет, хранит ли регистр историю:

| Периодичность | is_periodic | Поле Период в ДобавитьЗапись | Виртуальная таблица |
|---|---|---|---|
| `Непериодический` | false | Нет | — |
| `День` / `Месяц` / `Момент` | true | Да (`Период: Дата`) | `.СрезПоследних()` |

`.СрезПоследних()` возвращает строки с последним значением по каждому ключу измерений на указанную дату. Для непериодических регистров эта таблица не существует.

---

### Реквизиты vs Ресурсы

| | Ресурс | Реквизит |
|---|---|---|
| Входит в ключ? | Нет | Нет |
| Назначение | Хранимое числовое/строковое значение | Дополнительные атрибуты записи |
| В ДобавитьЗапись | Да | Да |
| В запросах | Как обычное поле | Как обычное поле |

Оба передаются в `ДобавитьЗапись` как именованные параметры — суффиксов нет.

## Строковая интерполяция в XBSL

| Синтаксис | Что делает |
|---|---|
| `${Выборка.Номенклатура}` | Вставляет строковое представление объекта (для ссылок — имя элемента) |
| `%{Выборка.Количество}` | Вставляет числовое значение |
| `%{-Выборка.КоличествоОстаток}` | Числовое значение со знаком минус — превращает отрицательный остаток в положительную нехватку |
| `\н` | Перенос строки |
