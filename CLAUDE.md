# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Назначение репозитория

Набор скиллов для AI-агентов, работающих с проектами на платформе **1С:Элемент (XBSL/SBSL)** — язык со статической типизацией для разработки бизнес-приложений.

## Структура репозитория

```
.claude/skills/          # скиллы Claude Code
    xbsl-uuid/           # генерация UUID v4
    xbsl-explore/        # разведка структуры проекта
        scripts/explore.py
    xbsl-meta-add/       # создание объектов конфигурации
        references/      # спецификации по типам объектов
tools/                   # материалы и инструменты (в .gitignore)
```

## Скиллы

### xbsl-uuid
Генерирует N UUID v4 через `python3`. Требует python3.

### xbsl-explore
Разведка структуры проекта 1С:Элемент. Запускает `scripts/explore.py`, возвращает `suggested_path` (куда класть новый файл) и `conflict` (конфликт имён). **Вызывается перед созданием любого объекта.**

### xbsl-meta-add
Создаёт объект конфигурации (`.yaml` + опционально `.xbsl`). Оркестрирует xbsl-explore и xbsl-uuid. Читает спецификацию из `references/{ТипОбъекта}.md`.

## Структура проекта 1С:Элемент

Проекты 1С:Элемент, с которыми работают скиллы, устроены так:
```
<корень>/
    <Проект>/
        Проект.yaml
        <Подсистема>/
            Подсистема.yaml
            СтатусЗадачи.yaml     # объект метаданных
            СтатусЗадачи.xbsl     # методы объекта (опционально)
```

Скрипт `explore.py` определяет проекты по наличию `Проект.yaml`, подсистемы — по `Подсистема.yaml`.

## Разработка скиллов

Каждый скилл — папка в `.claude/skills/` с обязательным файлом `SKILL.md`. Frontmatter:
```yaml
---
name: имя-скилла
description: >
  Описание для триггера скилла (по нему Claude решает, когда вызывать)
---
```

При разработке нового скилла для 1С:Элемент — ориентируйся на существующие скиллы как образец.
